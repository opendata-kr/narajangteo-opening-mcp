# data.go.kr 인증키 발급 가이드 (낙찰정보서비스)

> [!NOTE]
> 스텁이다. 첫 사용자용 **주석 스크린샷**(활용신청부터 Decoding 인증키 복사까지, `docs/images/`)은 발행 전 채운다.
> 캡처는 로그인 세션이 필요하므로 Chrome DevTools MCP로 만든다(리사이즈 1280×800 + DOM 오버레이 주석 → 스크린샷 저장).

## 핵심 (텍스트로 완결)

1. [공공데이터포털](https://www.data.go.kr)에 로그인한다(회원가입은 생략).
2. "**나라장터 낙찰정보서비스**"를 검색한다. 서비스 ID는 `ScsbidInfoService`, 제공기관은 조달청이다.
3. 오픈 API 상세에서 **활용신청**을 누른다. 활용목적을 적고, 필요 시 상세기능(오퍼레이션)을 선택하고, 라이선스 동의 후 신청한다.
4. 승인되면 마이페이지 > 오픈API > 개발계정에서 인증키를 확인한다.
5. **Decoding(원본) 인증키**를 복사해 `DATA_GO_KR_SERVICE_KEY`에 넣는다. Encoding 키를 넣으면 이중 인코딩으로 인증이 실패한다.

> [!IMPORTANT]
> 인증키는 계정당 하나지만, **각 API는 개별 활용신청 승인이 있어야 인증된다**. 다른 서비스에 쓰던 키라도 낙찰정보서비스 활용신청이 승인돼야 이 서버가 동작한다.
